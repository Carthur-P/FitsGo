About

This project was created for a client (Claire) as part of the Otago Polytechnic BIT course paper. This paper (Project - IN701) is design to help student feel confident about working in the IT field as apart of a team. Its aim is to simulate a real working environment by incouraging the student to work in a team to complete a large project task by a real client. This portfolio will showcase the work and troubles incountered throughout the whole development

The FitGo application is an excercise app which is aim to incourage people to excercise more by turning it into a fun game. The aim of the application is to get people to go outside and run/walk/bike around their city. The app features multiple interest points around Dunedin which the user can run to and collect their rewards. These stats can then be added to your rcord and profile to track your progression and compare with friends.

21/08/2019

Researching about react-native geofencing features that can be used to implement geofencing on the exercise application. Two possible ways of implementing geofencing were radius from a specified point and point in a polygon geofence. I decided to implement the point in a polygon geofence method which meant that we could customize the shape and size of the geofence to suit our requirement of the geofence being each suburbs in Dunedin. Using the radius from a specified point will mean that we could not customize the shape of the geofence to the shape we wanted, leading to a less accurate geofence. The attempt to implement geofencing in our application proved very difficult as multiple libraries were used which all failed. The most promising libraries that did not cause a fatal application error was called “react-native-geo-fencing”. A method in this library takes in two arguments, one being the actual point that is going to be tested, and two being an array of points that will form the geofence polygon. However, this method always returns a negative finding even though the point is in the polygon.

23/08/2019

Further research shows that many people have encountered the same problem as the library is outdated and has not been worked on for nearly two years. Other libraries were utilized to try and implement geofencing features into the app. However, all failed. Finally, I came across a thread which contained a self-written method that will take in two arguments similar to the method in the “react-native-geo-fencing” library and returns a Boolean value indicating whether the point is inside the set of points forming the polygon. This method was implemented into the code and geofencing functionality was achieved in the application.

Link to geofencing code
https://stackoverflow.com/questions/46265224/using-geofences-with-react-native

This forces the marker fetching function to be called only when the data is completely fetched from firebase.
